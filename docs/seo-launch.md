# SEO Launch Plan

## Decision

Publish canonical tutorial content on GitHub Pages and GitHub README first, with ClawHub as the install surface.

Why:

- The product is a developer tool, so GitHub has the strongest intent match.
- GitHub Pages can rank for long-tail queries such as `xai-org x-algorithm run Phoenix locally`.
- The canonical URL can point to the repo, ClawHub install surface, and the three-skill workflow.
- Medium can be used later as syndication, but should not be the canonical URL.
- Notion is useful as an internal launch note, but weak as the primary SEO surface.

Open Feed Recsys Lab: Verify, Audit, and Pretest X Algorithm Claims

## Meta Description

Generate commit-pinned reproducibility reports, source-backed claim audits, and bounded synthetic audience research for xai-org/x-algorithm.

## Publication Surfaces

- Canonical: GitHub Pages at `https://zack-dev-cm.github.io/open-feed-recsys-lab/`
- Source: GitHub repo
- Install: ClawHub `open-feed-recsys-lab`, `x-algo-claim-auditor`, and `tinytroupe-feed-research-lab`
- Video discovery: YouTube tutorial
- Syndication later: Medium and Notion import from `docs/tutorial-medium-notion.md`

## Do Not Claim

- Do not claim the public repo fully matches live production.
- Do not claim the tool predicts reach, virality, or ranking.
- Do not claim Phoenix was run unless extracted LFS artifacts are present.
- Do not claim the TinyTroupe-inspired skill is Microsoft TinyTroupe or a live X feed simulator.
